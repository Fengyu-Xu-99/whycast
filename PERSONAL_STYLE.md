# Personal Style Profile

WhyCast is currently a personal episode-making workflow, not a public app.

The goal is to create research-backed Chinese podcast scripts that the project owner personally wants to read first, then optionally hear as audio.

## Core Preference

Optimize for the owner's curiosity and taste before optimizing for a general audience.

The episode should feel like:

- a deep answer to a real personal question
- natural spoken Mandarin
- knowledge-first, not motivational
- specific, concrete, and research-backed
- readable before it becomes audio

The episode should not feel like:

- a generic educational summary
- a school essay
- an app demo
- a broad public content product
- a script filled with vague caveats
- a long meta reflection about curiosity itself

## Script Style

Use:

- short opening based on the real trigger
- direct entry into the question
- deeper explanation instead of quick overview
- concrete examples
- clear comparisons
- natural transitions
- casual but intelligent Simplified Chinese

Avoid:

- long scene-setting before the real question
- repeated "we need to be careful" narration
- vague phrases like "this may be helpful" without explaining how
- fluffy summary endings
- ending by explaining WhyCast too much
- forcing a fixed runtime if the question needs more depth

## Research Style

Research should happen before serious script drafting.

For each episode:

- use real sources
- separate strong claims from weaker background framing
- avoid fake precision
- do not include numbers unless the source is strong and the number matters
- keep source notes beside the script
- flag claims that need stronger sourcing before audio

The owner cares more about learning true things than about smooth-sounding but unsupported content.

## Feedback Learned From Bloodhound v1

What worked:

- shorter opening
- direct knowledge
- deeper explanation of body traits
- "smell as information" framing
- specific scent hound and sight hound examples
- comparison between tasks and body shapes

What to keep improving:

- stronger source notes
- deeper examples where the owner asks "how" or "why"
- more concrete history and origin when a category is introduced
- less generic summarizing at the end

## Default Episode Workflow

1. Capture the real trigger.
2. Turn the trigger into a core question.
3. Research before drafting.
4. Create an overview script.
5. Let the owner read it.
6. Revise based on what felt shallow, awkward, boring, or too vague.
7. Only create audio after the script feels worth listening to.

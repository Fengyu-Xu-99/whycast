# Roadmap

## Phase 0 - Repository Foundation

Goal: create a clear project home before building.

- Create North Star docs.
- Define MVP scope.
- Add Codex repo guidance in `AGENTS.md`.
- Keep a table-based change and decision log.
- Plan the first validation path.

## Phase 1 - Personal Episode Proof

Goal: prove the core idea manually for the project owner before building an app.

This phase answers one question:

> Can WhyCast turn one real curiosity spark into a research-backed Chinese podcast script that I personally want to read first, then listen to?

Work:

- Pick one curiosity seed.
- Collect 2-3 reference podcasts or episodes for style inspiration.
- Capture the owner's personal style preferences.
- Research the topic before drafting.
- Draft a podcast script.
- Revise based on owner feedback.
- Convert to audio only after the script feels worth listening to.

Deliverable:

- One complete test episode script.
- Source notes for the main claims.
- Optional generated audio draft.
- Notes on what worked, what felt boring, and what should change.

## Phase 2 - Personal Workflow System

Goal: turn the manual episode process into a repeatable personal workflow.

- Create a personal style profile.
- Create a reusable episode workflow.
- Create source-note conventions.
- Create a small evaluation checklist.
- Test prompts on 2-3 different topics.

## Phase 3 - Product Decision

Goal: decide whether WhyCast should become an app after several personal episodes exist.

- Review 2-3 completed personal episodes.
- Identify repeated steps.
- Decide whether to build a Codex skill, prompt pack, private script/audio pipeline, or app prototype.
- Update `PROJECT_SPEC.md` based on what was learned.

## Phase 4 - AI-Powered MVP

Goal: connect the proven personal workflow to software only if the workflow is repeatable.

- Generate grouped question maps from a seed.
- Generate episode outlines.
- Generate branch explanations or podcast script sections.
- Keep session state in the browser or simple backend storage.
- Add basic loading, error, and retry states.
- Add a small prompt/eval set for quality checks.

## Phase 5 - Usability Loop

Goal: make the product useful for weekly personal use.

- Save local sessions.
- Allow editing generated questions.
- Add regenerate controls by category.
- Add return-to-map flow.
- Add export to Markdown.
- Add optional audio generation flow.

## Future Possibilities

These should wait until the core loop is clearly useful:

- public app
- polished podcast generation
- documentary-style narration
- community maps
- graph database
- public sharing
- spaced repetition
- personalized recommendation engine

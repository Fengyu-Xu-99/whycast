# Sprint Plan

This plan keeps WhyCast small enough to learn from. The first milestone is not an app. The first milestone is one good personal episode workflow.

## Sprint 0 - Project Foundation

Goal: make the repo understandable and easy to continue.

Done when:

- `VISION.md` explains the North Star.
- `PROJECT_SPEC.md` explains the product scope.
- `ROADMAP.md` reflects the current validation path.
- `AGENTS.md` tells Codex how to work in this repo.
- `CHANGELOG.md` records meaningful changes and decisions.

Status: mostly done.

## Sprint 1 - Episode Proof Setup

Goal: define the first test episode before writing prompts.

Artifact:

- `episodes/bloodhound-test/episode-brief.md`

Tasks:

- Pick the first curiosity seed. Done: Bloodhound tracking, starting from *The Hunter: Call of the Wild*.
- Save 2-3 reference podcast links or names. Done: `碳基生物生存指南` and `原来是这样 Dscience`.
- Write what you like about their style. Done: warm science-backed storytelling and question-chain science explanation.
- Decide episode length target. Done: 15-20 minutes.
- Decide whether the first episode should be solo narration or host-style. Done: solo narration.

Done when:

- The first episode topic is selected.
- Style references are documented.
- The target episode format is clear.

Status: done.

## Sprint 2 - Curiosity Map Prompt

Goal: revise the Bloodhound script using the owner's personal style before creating audio.

Tasks:

- Capture the owner's script preferences in `PERSONAL_STYLE.md`.
- Create the repeatable episode workflow in `episodes/WORKFLOW.md`.
- Create a deeper Bloodhound script draft based on feedback.
- Keep source notes next to the draft.

Done when:

- The owner has a script draft that feels directionally right.
- The repo has a reusable workflow for the next episode.

## Sprint 3 - Research Notes And Fact Check

Goal: strengthen the Bloodhound episode's factual grounding before audio.

Tasks:

- Create `research-notes.md` for the Bloodhound episode.
- Tie major claims in the script to sources.
- Replace weak background sources where possible.
- Mark any claims that should be removed or softened.

Done when:

- The script is safe enough to learn from.

## Sprint 4 - Audio Draft

Goal: turn the approved script into a listenable audio draft.

Tasks:

- Choose text-to-speech approach.
- Add pronunciation and pacing notes.
- Generate a short audio sample.
- Revise script formatting for speech.
- Generate full audio only after the sample sounds acceptable.

Done when:

- One audio draft exists.
- There are clear notes for improving the next episode.

## Sprint 5 - Repeat On Two More Topics

Goal: test whether the workflow works beyond Bloodhound.

Tasks:

- Pick two more curiosity triggers from the owner's life.
- Run the same brief, research, script, feedback workflow.
- Compare what repeats across episodes.

Done when:

- There are 2-3 personal episodes or substantial drafts.
- Repeated workflow steps are clear.

## Sprint 6 - Build Decision

Goal: decide what, if anything, should be built after the personal workflow is proven.

Tasks:

- Review the manual episode workflow.
- Identify repeated steps.
- Decide whether to create a Codex skill, prompt pack, private tool, or app.
- Update `PROJECT_SPEC.md` based on what was learned.

Done when:

- The next build target is clear.
- Product scope is updated or intentionally deferred.
